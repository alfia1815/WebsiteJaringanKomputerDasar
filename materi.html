Jaringan Komputer Dasar — Materi Lengkap

Materi ini disusun untuk pemula, mencakup teori dan praktik. Baca tiap bagian secara berurutan untuk pemahaman terbaik.

1. Pengertian Jaringan Komputer

Jaringan komputer adalah kumpulan dua atau lebih perangkat komputer yang saling terhubung menggunakan media transmisi untuk berbagi data, sumber daya (seperti printer), dan layanan (seperti server). Koneksi bisa bersifat kabel (UTP, fiber optic) atau nirkabel (Wi-Fi).

Manfaat singkat:
  • Berbagi file & sumber daya
  • Kemudahan komunikasi
  • Manajemen terpusat (server)
  • Akses internet bersama

2. Komponen Utama Jaringan

  • Server — penyedia layanan (web, file, email).
  • Client — pengguna layanan (PC, laptop, smartphone).
  • Router — menghubungkan jaringan berbeda (mis. LAN ke WAN).
  • Switch — menghubungkan perangkat dalam satu jaringan lokal (LAN).
  • Access Point (AP) — menyediakan koneksi nirkabel.
  • Modem — mengubah sinyal ISP ke format yang bisa dipakai lokal.
  • Media — kabel UTP, coaxial, fiber optic; gelombang radio (Wi-Fi).
  • Protokol — aturan komunikasi seperti TCP/IP.

3. Topologi Jaringan

Topologi menjelaskan struktur koneksi antar perangkat. Contoh umum:

Topologi Star

Semua perangkat terhubung ke perangkat pusat (switch/hub). Mudah dipasang dan dikelola; kegagalan satu kabel tidak memengaruhi yang lain.

Topologi Bus

Semua perangkat berada pada satu kabel utama. Murah untuk jaringan kecil tetapi sulit skalanya dan rentan pada collision.

Topologi Ring

Perangkat tersusun melingkar; data mengalir ke satu arah (atau dua arah pada dual ring).

Topologi Mesh

Setiap perangkat dapat terhubung langsung ke perangkat lain — sangat fault-tolerant namun mahal.

4. Model OSI dan Fungsi Tiap Layer

Model OSI (Open Systems Interconnection) terdiri dari 7 layer. Ini membantu kita memahami di mana fungsi perangkat/protokol berada.

Layer Nama Fungsi Singkat Contoh Protokol / Perangkat
7ApplicationAntarmuka aplikasi pengguna (web, email)HTTP, SMTP, FTP
6PresentationFormat data, enkripsiSSL/TLS
5SessionManajemen sesi komunikasiNetBIOS, RPC
4TransportPengiriman data end-to-end, kontrol kesalahanTCP, UDP
3NetworkPengalamatan & routingIP, Router
2Data LinkTransfer frame antar nodeEthernet, Switch, MAC
1PhysicalTransmisi bit melalui mediaUTP, Fiber, Kabel

5. TCP/IP Stack (singkat)

Model yang praktis dipakai di Internet — terdiri dari Application, Transport, Internet, dan Link.

6. Alamat IP & Subnetting Dasar

Alamat IP (IPv4) terdiri dari 32 bit, biasanya ditulis empat oktet desimal (contoh: 192.168.1.10).

Contoh kelas IP (tradisional):
  • Kelas A: 1.0.0.0 – 126.255.255.255 (default subnet mask 255.0.0.0)
  • Kelas B: 128.0.0.0 – 191.255.255.255 (255.255.0.0)
  • Kelas C: 192.0.0.0 – 223.255.255.255 (255.255.255.0)
Subnet mask & CIDR

Contoh: 192.168.1.0/24 berarti mask 255.255.255.0 sehingga ada 256 alamat (254 host usable).

Catatan singkat: Untuk perhitungan subnet lebih jauh, pelajari cara konversi desimal-ke-biner, menghitung host per subnet, dan menentukan network/broadcast address.

7. Perangkat Jaringan & Fungsinya

Router

Menghubungkan 2+ jaringan berbeda (routing), bekerja pada Layer 3.

Switch

Menghubungkan perangkat dalam LAN, bekerja pada Layer 2 (dan L3 untuk managed switches).

Access Point

Menjadi titik akses nirkabel yang menghubungkan perangkat Wi-Fi ke jaringan kabel.

Modem

Mentranslasikan sinyal ISP (ADSL, cable, fiber) ke format digital untuk jaringan lokal.

Firewall

Perangkat lunak atau hardware untuk mengontrol trafik berdasarkan kebijakan keamanan.

8. Protokol Umum & Peranannya

  • HTTP / HTTPS: protokol untuk web.
  • TCP: koneksi yang andal (handshake, retransmit).
  • UDP: koneksi tanpa jaminan (video, VoIP).
  • DNS: menerjemahkan nama domain ke IP.
  • DHCP: otomatis memberikan alamat IP ke perangkat.

9. Keamanan Jaringan Dasar

Keamanan penting dari desain hingga operasi. Beberapa langkah awal:

  1. Gunakan password kuat pada perangkat jaringan.
  2. Segmentasi jaringan (VLAN) untuk isolasi trafik.
  3. Aktifkan firewall & update firmware/perangkat lunak.
  4. Gunakan enkripsi (WPA2/WPA3 untuk Wi-Fi, TLS untuk aplikasi).
  5. Monitor trafik dan buat log aktivitas.

10. Praktik Dasar (Perintah & Contoh)

Perintah ping (uji konektivitas)
ping 8.8.8.8
ping google.com
Perintah melihat konfigurasi IP

Windows:

ipconfig /all

Linux / macOS:

ifconfig
atau
ip addr show
Traceroute / Tracert (lihat jalur paket)
tracert google.com   (Windows)
traceroute google.com (Linux/macOS)
Contoh konfigurasi singkat (conceptual)

Jika ingin membuat jaringan kecil (kelas lab):

  • Router: konfigurasi interface WAN (dari ISP) dan LAN (mis. 192.168.10.1/24)
  • Switch: hubungkan PC dan server ke switch
  • Server DHCP: aktifkan untuk mengalokasikan IP otomatis pada subnet 192.168.10.0/24

11. Contoh Kasus & Solusi Singkat

Kasus: Tidak ada koneksi internet padahal kabel terpasang
  1. Cek lampu indikator modem/router
  2. Ping ke gateway (misal 192.168.1.1)
  3. Periksa konfigurasi IP (pastikan tidak duplicate)
  4. Restart modem/router & device
Kasus: jaringan lambat
  • Periksa penggunaan bandwidth (aplikasi intensif seperti streaming)
  • Periksa kualitas sinyal Wi-Fi (ganti kanal jika banyak interferensi)
  • Gunakan QoS untuk prioritas trafik

12. Lanjutan & Sumber Belajar

Jika ingin mendalami: pelajari subnetting mendalam, routing protocol (OSPF, BGP), VLAN, switching advanced, serta cybersecurity. Tools latihan: Cisco Packet Tracer, GNS3, atau lab fisik.

Rekomendasi bacaan/latihan:

  1. Dokumentasi resmi TCP/IP & RFC
  2. Materi kursus CCNA (dasar routing & switching)
  3. Praktik di lab virtual
